Automatic Download Schedule

Managed by GitHub Actions -- runs in the cloud every hour
🔁 Current schedule: every hour
GitHub Actions runs download_data.ps1 automatically every hour (00:00, 01:00, 02:00 … UTC), commits any new ThingSpeak data to the repo, and deploys it to GitHub Pages. Your PC does not need to be on.
🔗 View run history & trigger manually ✎ Edit schedule (change cron)
Changing the frequency
Click Edit schedule above to open the workflow YAML on GitHub. Change the cron line:
cron: '0 * * * *' Common alternatives: '0 */6 * * *' (every 6 h)  |  '0 8 * * *' (daily at 08:00 UTC)

Save & Apply Settings

Save locally and generate the config file for download scripts
How settings are applied
Settings you save here are stored in your browser (localStorage) and used by the dashboard automatically. To use them with download_data.ps1 on your local machine, click Download config.json and save the file next to download_data.ps1 in the repo folder. GitHub Actions does not read config.json — it uses the channel defaults built into the script.
✓ Settings saved!

Local Development

Run download scripts manually on your own machine
config.json is still used for local runs
When running download_data.ps1 directly on your PC, it reads config.json from the same folder. Use Save to Browser then Download config.json below, and save the file next to download_data.ps1 in the repo folder. GitHub Actions does not use this file — it uses the defaults built into the script.
Run a manual download any time
Open PowerShell in the repo folder and run: .\download_data.ps1 Or with override options: .\download_data.ps1 -ScheduleType hourly -MaxResults 8000
